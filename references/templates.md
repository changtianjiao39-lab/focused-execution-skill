# Response Templates

Use the smallest template that fits. Keep the output compact enough to act on immediately.

## Launch card

```markdown
## 本轮结论
[一句话决策]

## 唯一主任务
- 交付物：
- 截止时间：
- 本轮质量：最低 / 可用 / 理想
- 验收标准：

## 三遍顺序
1. 骨架：
2. 可用：
3. 优化（暂不做）：

## 现在开始的 10 分钟成果
[一个可见产物]

## 专注保护
- 如果……那么……
- 暂不处理：
- 第一个证据检查点：
- Plan B（缩小范围）：
```

## Switching decision

```markdown
## 切换判断：继续当前任务 / 允许切换
- 当前任务的影响与截止：
- 新任务的真实后果：
- 是否阻塞当前任务：
- 切换成本：
- 结论：

若不切换：把新想法写成一行停车项。
若切换：先记录做到哪里、回来后的下一步、恢复所需资料。
```

## Rescue plan

```markdown
## 卡点类型
[一个卡点类别]

## 不需要推翻的部分
[保留仍然有效的工作]

## 最小修复动作
[一个动作或测试]

## 降级后的可交付版本
[缩小但诚实的范围]

## 下一次证据检查
[时间或产物]
```

## Session review

```markdown
## 实际证据
- 产生的成果：
- 启动延迟：
- 主动切换次数：
- 被什么带走：
- 是否发生范围扩张：
- 是否产生熬夜成本：

## 下一轮只调整一件事
[一个干预变量]

## 恢复点
[准确的下一步]
```
