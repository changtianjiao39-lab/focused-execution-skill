# Coaching Patterns

Use this reference to match observed behavior to a small intervention. Ask for evidence from the current task instead of assuming a stable personal trait.

## Pattern matrix

| Observed pattern | Useful interpretation | First intervention | Avoid |
|---|---|---|---|
| Starts only when the deadline feels dangerous | Urgency may be acting as the start signal | Create an earlier artifact checkpoint with a named reviewer or acceptance test | Adding only another vague deadline |
| Planning, research, or organization keeps expanding | Preparation may be replacing exposure to uncertain core work | Cap planning; name the decision each extra step would change; produce a 10-minute artifact | Building a larger productivity system |
| Easy tasks repeatedly beat important work | Ease is being used as a priority signal | Rank by consequence, impact, and dependency; allow only direct unblockers | A warm-up list of unrelated quick wins |
| Every new idea feels more important | Novelty or unresolved concern is claiming priority | Use a one-line parking lot and the switching gate | Opening a new tab, document, or task immediately |
| Details are polished before the whole exists | Local control may feel safer than incomplete structure | Require skeleton, usable version, then selective polish | Calling perfectionism a personality flaw |
| An interruption makes it hard to return | The unfinished task may leave attention residue | Write a ready-to-resume note before switching | Trusting memory to recover context |
| One setback triggers a restart or full redesign | The gap from the ideal may be obscuring valid work | Preserve what remains true; run the smallest test; reduce scope if needed | Throwing away the whole plan |
| Low energy is treated as moral failure | Capacity and task design may be mismatched | Shorten the block, reduce the deliverable, protect sleep, then observe | Shame, all-nighters, or diagnostic claims |

## Evidence questions

Use only the questions needed to select the next action:

- What concrete artifact must exist when this is done?
- What did the person do instead of the main task?
- At what moment did the scope expand or attention switch?
- Which decision would more research actually change?
- Is the obstacle uncertainty, skill, information, energy, emotion, or dependency?
- What portion is already valid and should be preserved?
- What is the smallest visible proof of progress within 10 minutes?

## Strengths to preserve

The same pattern can contain an asset:

- divergent thinking can reveal dependencies and alternatives;
- high standards can improve selective polish;
- detailed thinking can expose risk;
- deadline performance can show capacity under pressure;
- reflection can support deliberate experiments.

Redirect strengths by stage: diverge during exploration, converge during execution, and apply high standards only after a usable whole exists.

## Intervention sequence

When several patterns appear together, use this order:

1. define the deliverable and quality level;
2. select one main task;
3. create visible evidence quickly;
4. protect it with one if-then rule;
5. install an early checkpoint;
6. review actual switches, artifacts, and sleep cost.

Do not add every intervention at once. Start with the smallest one that targets the observed bottleneck, then change one variable per review.
